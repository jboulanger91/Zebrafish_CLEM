# tests

Regression and integration tests for the classifier prediction pipeline.

## Overview

This package contains regression tests that verify the pipeline produces identical outputs before and after code changes. The test suite operates in two phases: first, baseline values are captured by running the full pipeline and saving key metrics to JSON files; then, pytest tests compare current pipeline outputs against those saved baselines.

## Files

| File | Description |
|------|-------------|
| `conftest.py` | Pytest configuration and shared fixtures. Session-scoped fixtures run the full pipeline once and share results across all tests. **Pipeline fixtures:** `pipeline_results`, `data`, `rfe_result`, `cv_result`, `predictions`. **Step-by-step fixtures:** `predictor_after_load_cells`, `predictor_after_load_features`, `predictor_after_preprocessing`, `predictor_after_rfe`, `predictor_after_confusion_matrices`, `predictor_after_predictions`, `predictor_after_verification`. **Reference data fixtures:** `clem_reference_predictions`, `em_reference_predictions`. |
| `test_pipeline_regression.py` | Main regression tests (~1100 lines). Container-returning tests: `TestLoadData`, `TestFeatureSelection`, `TestCrossValidation`, `TestPredictionContainers`, `TestVerification`, `TestEndToEnd`. Attribute-based tests: `TestLoadCellsDF`, `TestCalculateMetrics`, `TestPreprocessing`, `TestRFEFeatureSelection`, `TestConfusionMatrices`, `TestPredictions`, `TestVerificationMetrics`, `TestEdgeCases`, `TestDataIntegrity`, `TestCrossReference`. |
| `test_detailed_regression.py` | Detailed regression tests (~550 lines). Coverage of intermediate values: confusion matrix values, probability statistics, scaled predictions, feature matrix shape, outlier detection counts, DataFrame columns, preprocessing effects, training data consistency, specific cell predictions. |
| `test_segclr_integration.py` | Integration tests for SegCLR embedding loading (`load_segclr_embeddings`) and combined feature classification (`combine_features`). Uses synthetic HDF5 fixtures. |
| `test_verification_metrics.py` | Standalone verification metrics analysis (run as `__main__`). Computes accuracy and F1 matrices across all combinations of verification variables (OCSVM, IF, LOF, intra-class, NBLAST). Saves heatmap PDFs to `classifier_pipeline/test_verification_metrics/`. |
| `capture_baseline.py` | Captures basic baseline values (7-stage pipeline) to `baseline_values.json`. |
| `capture_detailed_baseline.py` | Captures detailed baseline values (6-stage pipeline) to `detailed_baseline_values.json`. |
| `run_tests.py` | CLI test runner. Supports `--capture-baselines`, `--all`, `--basic`, `--detailed`, `--class TestName`, `-v`. |

## Key Test Classes

**Container-returning methods:**
- `TestLoadData` -- `load_data()` returns `LoadedData` with correct training counts and modality masks
- `TestFeatureSelection` -- `select_features_rfe()` returns `RFEResult` with correct feature count
- `TestCrossValidation` -- `cross_validate()` returns `CVResult` with 4x4 confusion matrix
- `TestPredictionContainers` -- `predict()` returns `PredictionResults` with correct counts and distribution
- `TestVerification` -- `verify()` sets verification status with correct pass counts

**Attribute-based methods:**
- `TestLoadCellsDF` -- Cell count, modality distribution, function distribution, no duplicates
- `TestRFEFeatureSelection` -- Feature count, RFE curve, exact selected feature names
- `TestConfusionMatrices` -- CM values, F1 score
- `TestPredictions` -- Prediction counts, cell-by-cell comparison to CLEM and EM reference Excel files
- `TestVerificationMetrics` -- Pass counts (total, EM, CLEM), comparison to reference files
- `TestEdgeCases` -- No NaN in predictions/probabilities, valid class names, valid modalities
- `TestDataIntegrity` -- No infinite/NaN features, labels match function column, exclusive modality indices

## Baseline Files

- `baseline_values.json` -- Basic pipeline metrics (generated by `capture_baseline.py`)
- `detailed_baseline_values.json` -- Detailed intermediate values (generated by `capture_detailed_baseline.py`)

These JSON files must be regenerated whenever the pipeline's expected behavior intentionally changes.

## Dependencies

- **`core/`** -- `ClassPredictor`, container types, `AdaBoostClassifier`, `LinearDiscriminantAnalysis`
- **`src/util/`** -- `get_base_path()`, `get_output_dir()`
- External: pytest, scikit-learn, pandas, numpy, matplotlib

## How to Run

```bash
# Capture baselines (run once before refactoring)
python tests/run_tests.py --capture-baselines

# Run all regression tests
python tests/run_tests.py --all -v

# Run only basic regression tests
python tests/run_tests.py --basic -v

# Run only detailed regression tests
python tests/run_tests.py --detailed -v

# Run a specific test class
python tests/run_tests.py --class TestConfusionMatrixValues -v

# Run container-returning tests only
pytest tests/test_pipeline_regression.py -v -k "TestLoadData or TestFeatureSelection or TestCrossValidation"
```
